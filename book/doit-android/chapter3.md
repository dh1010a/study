# chapter 3. 코틀린 시작하기

## 3-1 코틀린 언어 소개
- 코틀린은 JVM 기반으로 동작함
  - `코틀린 소스(.kt) -> 코틀린 컴파일러(kotlinc) -> 자바 바이트 코드 -> 자바 가상 머신`
- 코틀린으로 앱을 개발하면 자바보다 어떤 이점이 있는지?
  - 표현력과 간결함
  - 안전한 코드: null safety 지원. 객체 지향은 null을 허용하지만 코틀린은 널 허용과 불허용을 구분해서 선언함 
  - 상호 운용성: 코틀린은 자바와 100퍼센트 호환됨. 자바의 라이브러리와 클래스를 이용할 수 있고, 반대도 가능함. 하나의 앱을 개발할 때도 혼용해도 됨.
  - 구조적 동시성 : 코루틴 기법을 이용하면 비동기 프로그래밍을 간소화 할 수 있음. 더 간단하고, 효율적으로
- 코틀린 패키지 구문은 이 파일을 컴파일 했을 때 만들어지는 클래스 파일의 위치를 나타냄. 다만 kt 파일의 위치와 상관 없는 별도의 이름으로 선언 가능함
- 컴파일 된 클래스 파일은 해당 패키지에 생성됨
- 결국 코틀린 파일명과 클래스명은 아예 다르게 선언해도 상관 없음. 객체 지향만을 목적으로 한 언어가 아니기 때문
- 코틀린은 변수와 함수 등을 클래스로 묶지 않고 최상위에 선언할 수 있으므로 파일명과 클래스명은 상관 없음

### 변수와 함수
- 변수는 val, var로 선언
- val은 value. 초기값이 할당되면 바꿀 수 없음
- var은 variable. 초기값이 할당된 후에도 값을 바꿀 수 있는 변수를 선언할 때 사용
- 변수명 뒤에는 콜론(:)을 추가해 타입을 명시할 수 있으며, 대입하는 값에 따라 타입을 유추(타입 추론)할 수 있을 떄는 생략할 수 있음
- 최상위에 선언한 변수나 클래스의 멤버 변수는 선언과 동시에 초기값을 할당해야 하며, 함수 내부에 선언한 변수는 선언과 동시에 초깃값을 할당하지 않아도 됨. 물론 변수를 이용하려면 값을 할당하고 이용해야 함
- 변수를 선언할 때 초깃값을 할당할 수 없는 경우가 있음. 이때는 lateinit이나 lazy 키워드를 이용함
- lateinit 키워드는 이후에 초깃값을 할당할 것임을 명시적으로 선언함. 
  - lateinit은 var 키워드로 선언함 변수에만 사용할 수 있음
  - Int, Long, Short, Double, Float, Boolean, Byte 타입에는 사용할 수 없음
  - `lateinit var data1: String`
- lazy 키워드는 변수 선언문 뒤에 `by lazy{}` 형식으로 선언하며, 소스에서 변수가 최초로 이용되는 순간 중괄호로 묶은 부분이 자동으로 실행되어 초기 할당됨. 여러줄로 작성한다면 마지막 줄의 실행 결과가 변수의 초기값이 됨

### 데이터 타입
- 코틀린의 모든 변수는 객체. 따라서 코틀린의 모든 타입은 객체 타입
- 정수를 다루는 타입이 Int인데 Int는 기초 데이터 타입이 아니라 클래스다.
- 기초 타입 객체 : 정수를 표현하는 Int, Short, Long / 실수를 표현하는 Double, Float / 2진수를 표현하는 Byte / true나 false를 표현하는 Boolean 타입이 있음
- 문자와 문자열 : Char는 문자를 표현하는 타입. 코틀린 코드에서 Char 타입의 데이터는 문자를 작은따옴표(')로 감싸서 표현하며 Number 타입으로 표현할 수 없음
- Any: 코틀린에서 최상위 클래스. 모든 클래스는 Any의 하위 클래스이며 Any 타입으로 선언한 변수에는 모든 타입의 데이터를 할당할 수 있음
- Unit: 반환문이 없는 함수. 다른 타입과 다르게 데이터의 형식이 아닌 특수한 상황을 표현하려는 목적으로 사용함. 해당 타입으로 선언한 변수에는 Unit 객체만 대입할 수 있음. 따라서 변수를 선언할 수는 있지만 의미가 없고, 이런 Unit은 주로 함수의 반환타입으로 사용해 함수에서 반환문이 없음을 명시적으로 나타낼 때 사용
- Noting: null이나 예외를 반환하는 함수. Nothing으로 선언한 변수에는 null만 대입할 수 있으며, 데이터로서 의미가 없음. 주로 함수 반환타입에 사용하며 반환은 하는데 의미가 있는 값을 반환하지 않다는 뜻이나 예외를 던지는 함수의 반환타입을 선언할떄 사용

### 함수 선언하기
- `fun 함수명(매개변수명: 타입): 반환 타입 { ... }`
- 반환 타입을 생략하면 자동으로 Unit타입 적용
- 함수의 매개변수에는 var이나 val 키워드를 사용할 수 없음. val이 자동으로 적용되며 함수 안에서 매개변수 값을 변경할 수 없음
- 함수 매개변수에는 기본값을 선언할 수 있음. 기본값 선언시 호출할 때 인자를 전달하지 않아도 됨
- 매개변수가 여러 개면 호출할 때 전달한 인자를 순서대로 할당함. 즉 첫번째 인자를 첫 번째 매개변수에 할당함. 호출할 때 매개변수명을 지정하면 순서를 바꿔도 됨. 이것을 명명된 매개변수라고 함

### 컬렉션 타입
- Array클래스의 생성자에서 첫 번째 매개변수는 배열의 크기, 두 번째 매개변수는 초기값 지정. 배열의 타입은 제네릭으로 표현함. 
- 배열 데이터가 기초 타입이라면 Array를 사용하지 않고 각 기초 타입의 배열을 나타내는 클래스를 쓸 수 있음
  - ByteArray, ShortArray, IntArray, LongArray, FloatArray, DoubleArray, CharArray, BooleanArray
- arrayOf()라는 함수를 이용하면 배열을 선언할 때 값을 할당할 수 있음. `arrayOf<Int>(10, 20, 30)`. booleanArrayOf() 등도 사용 가능
- List, Set, Map은 Collection 인터페이스를 타입으로 표현한 클래스이며 통틀어 컬렉션 타입 클래스라고 함
- List는 불변클래스라서 size(), get()만 제공하고 add(), set() 함수는 제공하지 않음. MutableList는 가변 타입이므로 수정할 수 있음
- Set, Map도 마찬가지로 불변이며 MutableSet, MutableMap은 가변 타입임.
  - 안정성(데이터 변경 불가), 설계 의도 표현, 다형성 (외부에는 읽기 전용으로만 노출, 내부에서는 수정 가능)


### 조건문/반복문
- if else를 결괏값을 반환하는 표현식으로도 사용할 수 있음
- when 조건문도 존재. case 문과 비슷. 표현식으로 사용 가능
- for 조건에는 in 범위 연산자 사용
- until(끝 미포함)/step(원하는만큼 증가)/downTo(감소)
- indices는 컬렉션 타입의 인덱스 값을 의미하므로 포문에서 돌릴 수 있음. 값을 가져오려면 withIndex()를 이용함
